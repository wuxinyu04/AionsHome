# AionsHome 微信接入部署教程

本文说明如何让每个部署 AionsHome 的用户，把自己的微信绑定到自己的 AionsHome 实例里。

AionsHome 当前使用 OpenClaw 的 `openclaw-weixin` 微信插件完成扫码登录和 iLink 凭据保存，但消息上下文、绑定关系、AI 回复、前端展示仍然全部由 AionsHome 自己处理。也就是说，微信只是一个额外消息通道，不会替代 AionsHome 的主聊天窗口。

参考资料：

- OpenClaw WeChat 官方文档：https://docs.openclaw.ai/channels/wechat
- Tencent `openclaw-weixin` 插件仓库：https://github.com/Tencent/openclaw-weixin

## 功能效果

接入后：

1. 用户可以从微信给 AionsHome 发消息。
2. AionsHome 会把微信消息插入原本的聊天窗口，内容前缀为：

   ```text
   本条消息来自于微信：...
   ```

3. AI 可以在自己的回复里使用：

   ```text
   [微信消息：内容]
   ```

4. AionsHome 会隐藏这条工具指令，但会插入一条可见、且会进入上下文的系统消息：

   ```text
   本条为微信消息：内容
   ```

5. 在聊天室里，微信收到的消息会自动带发送者名字前缀，例如：

   ```text
   Aion：...
   Connor：...
   ```

   实际名字会读取当前项目配置，不需要写死。

## 一、准备条件

部署者需要准备：

1. 一套能正常启动的 AionsHome。
2. 一部可以扫码确认登录的微信。
3. 本机安装 Node.js/npm。
4. 本机安装 OpenClaw CLI，命令行里能执行：

   ```bash
   openclaw --version
   ```

5. AionsHome 后端正在运行，默认地址是：

   ```text
   http://127.0.0.1:8080
   ```

如果你的 AionsHome 后端端口不是 `8080`，下面所有命令里的端口都要替换成你的实际端口。

## 二、安装 OpenClaw 微信插件

先检查 OpenClaw 版本：

```bash
openclaw --version
```

官方插件兼容关系大致是：

- `@tencent-weixin/openclaw-weixin` 最新版：OpenClaw `>= 2026.3.22`
- `@tencent-weixin/openclaw-weixin@legacy`：OpenClaw `>= 2026.1.0 < 2026.3.22`

推荐优先使用快速安装：

```bash
npx -y @tencent-weixin/openclaw-weixin-cli install
```

如果快速安装失败，可以手动安装：

```bash
openclaw plugins install "@tencent-weixin/openclaw-weixin"
openclaw config set plugins.entries.openclaw-weixin.enabled true
```

如果提示 OpenClaw 版本太旧，可以安装 legacy 版本：

```bash
openclaw plugins install "@tencent-weixin/openclaw-weixin@legacy"
openclaw config set plugins.entries.openclaw-weixin.enabled true
```

## 三、扫码登录微信

在运行 AionsHome 的同一台机器上执行：

```bash
openclaw channels login --channel openclaw-weixin
```

终端会出现二维码。用手机微信扫码并确认授权。

登录成功后，插件会把账号凭据保存在本机 OpenClaw 状态目录中，默认位置通常是：

```text
~/.openclaw/openclaw-weixin/accounts
```

Windows 对应：

```text
C:\Users\你的用户名\.openclaw\openclaw-weixin\accounts
```

这些文件里包含本机微信通道凭据，不能提交到 Git，也不能发给别人。

## 四、启用 AionsHome 微信桥接

确认 AionsHome 后端已经启动后，打开一个新的终端。

Windows PowerShell：

```powershell
Invoke-RestMethod `
  -Method Put `
  -Uri "http://127.0.0.1:8080/api/settings/wechat-bridge" `
  -ContentType "application/json" `
  -Body '{"enabled":true,"transport":"openclaw"}'
```

macOS/Linux：

```bash
curl -X PUT "http://127.0.0.1:8080/api/settings/wechat-bridge" \
  -H "Content-Type: application/json" \
  -d '{"enabled":true,"transport":"openclaw"}'
```

如果你的 OpenClaw 状态目录不是默认的 `~/.openclaw`，可以显式指定：

Windows PowerShell：

```powershell
Invoke-RestMethod `
  -Method Put `
  -Uri "http://127.0.0.1:8080/api/settings/wechat-bridge" `
  -ContentType "application/json" `
  -Body '{"enabled":true,"transport":"openclaw","openclaw_home":"D:\\OpenClawState"}'
```

macOS/Linux：

```bash
curl -X PUT "http://127.0.0.1:8080/api/settings/wechat-bridge" \
  -H "Content-Type: application/json" \
  -d '{"enabled":true,"transport":"openclaw","openclaw_home":"/home/you/.openclaw"}'
```

检查状态：

Windows PowerShell：

```powershell
Invoke-RestMethod "http://127.0.0.1:8080/api/settings/wechat-bridge" |
  ConvertTo-Json -Depth 8
```

macOS/Linux：

```bash
curl "http://127.0.0.1:8080/api/settings/wechat-bridge"
```

重点看：

- `wechat_bridge_enabled` 是否为 `true`
- `wechat_bridge_transport` 是否为 `openclaw`
- `openclaw_accounts` 是否能看到账号
- `openclaw_status_error` 是否为空

`openclaw_accounts` 只会返回账号摘要，例如 `account_id`、`user_id`、`has_token`、`known_contacts`，不会把 token 明文返回给前端。

## 五、绑定微信到聊天窗口

### 推荐方式：自动绑定最后活跃窗口

这是最省事的方式。

1. 在 AionsHome 前端打开你想绑定的聊天窗口。
2. 如果想绑定聊天室，就先打开那个聊天室并让它成为最近活跃窗口。
3. 如果想绑定私聊，就先打开对应私聊窗口并让它成为最近活跃窗口。
4. 在微信里找到刚扫码登录的 OpenClaw/ClawBot 会话，发送：

   ```text
   绑定 AionsHome
   ```

   英文也可以：

   ```text
   bind AionsHome
   ```

5. AionsHome 会自动把这条微信会话绑定到最近活跃的 AionsHome 窗口。

如果最近活跃的是聊天室，就绑定聊天室；如果最近活跃的是私聊，就绑定私聊。之后这个微信会话发来的消息会进入对应窗口。
这里需要注意的是，如果重启了服务器，默认最后活跃的窗口可能会回到私聊，如果想要绑定聊天室，需要先在聊天室进行一次对话激活。

### 备用方式：生成绑定码

如果你想用绑定码，也可以调用接口创建一次性绑定码。

Windows PowerShell：

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8080/api/settings/wechat-bridge/bindings" `
  -ContentType "application/json" `
  -Body '{}'
```

macOS/Linux：

```bash
curl -X POST "http://127.0.0.1:8080/api/settings/wechat-bridge/bindings" \
  -H "Content-Type: application/json" \
  -d '{}'
```

返回里会有：

```json
{
  "code": "ABC123",
  "instruction": "Send this in WeChat: bind ABC123"
}
```

然后在微信里发送：

```text
bind ABC123
```

或：

```text
绑定 ABC123
```

绑定码默认 10 分钟有效。

如果接口提示 `source_type and source_id are required when no recent WeChat route exists`，说明当前没有可用的最近路由。优先使用上一节的“绑定 AionsHome”自动绑定方式，或者让前端先进入目标聊天窗口后再试。

## 六、测试收发

### 测试微信发到 AionsHome

在微信里发：

```text
测试一下
```

AionsHome 前端对应窗口应该出现一条用户消息：

```text
本条消息来自于微信：测试一下
```

这条消息会作为普通用户消息进入上下文，AI 后续能知道用户刚才是从微信来的。

### 测试 AionsHome 发到微信

在 AionsHome 里让 AI 使用能力：

```text
[微信消息：我在微信上提醒你一下]
```

预期效果：

1. 这段工具指令不会显示在 AI 普通聊天气泡里。
2. AionsHome 前端会出现系统消息：

   ```text
   本条为微信消息：我在微信上提醒你一下
   ```

3. 微信会收到对应内容。
4. 如果是在聊天室里，微信内容会带 AI 名字前缀，例如：

   ```text
   Aion：我在微信上提醒你一下
   ```

## 七、OpenClaw Gateway 要不要启动？

普通 AionsHome 用户不需要让 OpenClaw Gateway 长期接管微信消息。

本项目的做法是：

1. 用 `openclaw-weixin` 插件完成扫码登录。
2. AionsHome 读取本机 OpenClaw 保存的微信账号凭据。
3. AionsHome 自己轮询微信消息、写入自己的数据库、构建自己的上下文、发送自己的回复。

如果你同时启动 OpenClaw Gateway，它可能会把微信消息交给 OpenClaw 自己的默认 agent 处理，导致微信里出现类似：

```text
Missing API key for provider "openai".
```

这不是 AionsHome 微信桥接缺 OpenAI key，而是 OpenClaw 自己的 agent 在回复。AionsHome 只做微信通道时，不需要为了这个错误去配置 OpenClaw 的 OpenAI key。

如果你遇到这种情况：

1. 停掉 OpenClaw Gateway。
2. 保持 AionsHome 后端运行。
3. 确认 `/api/settings/wechat-bridge` 里 `transport` 是 `openclaw`。
4. 再从微信发送消息测试。

## 八、多人部署和开源注意事项

这个功能适合开源项目使用，因为每个用户都在自己的机器上完成自己的绑定。

不要把以下文件提交到 Git：

- `aion-chat/data/settings.json`
- `aion-chat/data/chat.db`
- `aion-chat/data/worldbook.json`
- `~/.openclaw/openclaw-weixin/accounts`
- 任何包含 token、context token、聊天记录、用户画像的文件

项目根目录 `.gitignore` 已经忽略了 AionsHome 的主要本地数据文件。OpenClaw 的 `~/.openclaw` 通常在用户主目录，不在项目仓库里，也不要手动复制进仓库。

每个用户的部署流程都是：

1. 拉取项目代码。
2. 安装 AionsHome 依赖并启动后端。
3. 安装 OpenClaw 微信插件。
4. 用自己的微信扫码登录。
5. 启用 AionsHome 微信桥接。
6. 在微信里发送 `绑定 AionsHome`。

这样不会共享你的微信，也不会共享你的绑定关系。

## 九、常见问题

### 1. `Invoke-RestMethod` 提示无法连接远程服务器

通常是 AionsHome 后端没启动，或者端口不是 `8080`。

先确认：

```powershell
Invoke-RestMethod "http://127.0.0.1:8080/api/settings/wechat-bridge"
```

如果仍然连不上，检查启动脚本、后端日志和实际端口。

### 2. `openclaw_accounts` 是空数组

常见原因：

1. 还没有执行扫码登录。
2. 扫码登录不是在运行 AionsHome 的同一台机器上完成的。
3. OpenClaw 状态目录不是默认位置，但 AionsHome 没配置 `openclaw_home`。
4. 插件安装失败或版本不兼容。

可以检查：

```bash
openclaw --version
openclaw plugins list
openclaw channels login --channel openclaw-weixin
```

### 3. 微信发了消息，但 AionsHome 没反应

按顺序检查：

1. AionsHome 后端是否一直在运行。
2. `/api/settings/wechat-bridge` 里的 `wechat_bridge_enabled` 是否为 `true`。
3. `wechat_bridge_transport` 是否为 `openclaw`。
4. 微信会话是否已经发送过 `绑定 AionsHome` 或 `bind <code>`。
5. 当前微信会话是否和绑定时的会话一致。

如果从群聊发消息，是否能收到还取决于当前微信插件和账号实际开放的群聊能力。OpenClaw 官方文档说明当前插件元数据主要标注了私聊和媒体能力，群聊能力可能不会在插件元数据里明确声明。

### 4. AI 发了 `[微信消息：...]`，微信没收到

检查：

1. 这个聊天窗口是否已经绑定微信。
2. 设置里 `transport` 是否是 `openclaw`。
3. `openclaw_accounts` 是否能看到登录账号。
4. 后端日志里是否有 `[WECHAT_BRIDGE]` 相关错误。

如果没有绑定，AionsHome 仍然会在本地插入系统消息，但不会真正发到微信。

### 5. 微信里收到 `Missing API key for provider "openai"`

这是 OpenClaw Gateway 的默认 agent 在回复，不是 AionsHome 缺 key。

如果你只想使用 AionsHome 微信桥接，请停止 OpenClaw Gateway，让 AionsHome 后端接管微信轮询和回复。

### 6. 想重新绑定到另一个窗口

打开新的目标聊天窗口，让它成为最近活跃窗口，然后在微信里重新发送：

```text
绑定 AionsHome
```

同一个微信会话重新绑定时，会覆盖旧绑定。

### 7. 想关闭微信桥接

Windows PowerShell：

```powershell
Invoke-RestMethod `
  -Method Put `
  -Uri "http://127.0.0.1:8080/api/settings/wechat-bridge" `
  -ContentType "application/json" `
  -Body '{"enabled":false}'
```

macOS/Linux：

```bash
curl -X PUT "http://127.0.0.1:8080/api/settings/wechat-bridge" \
  -H "Content-Type: application/json" \
  -d '{"enabled":false}'
```

## 十、给维护者的检查清单

发布前建议确认：

1. `.gitignore` 没有放行任何本地隐私数据。
2. 新用户文档里没有你的账号 id、微信 user id、context token。
3. `aion-chat/data/settings.json` 没有被提交。
4. `aion-chat/data/chat.db` 没有被提交。
5. AionsHome 后端启动后，`/api/settings/wechat-bridge` 能正常返回。
6. 从微信发送消息可以进入 AionsHome 前端。
7. AI 使用 `[微信消息：...]` 后，微信能收到消息，前端也能看到 `本条为微信消息：...` 系统记录。
